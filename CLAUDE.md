# CLAUDE.md

## 🌟 스터디 목표
- "생각한 것을 바로 코드로!" (Vibe Coding)
- 복잡한 이론보다 '동작하는 서비스'를 우선합니다.
- 비개발자 팀원도 이해할 수 있는 쉬운 코드를 지향합니다.

## 🛠 기술 스택 (Claude가 참고할 것)
- 프레임워크: Next.js (App Router)
- 스타일: Tailwind CSS + shadcn/ui (예쁜 디자인을 위해 필수!)
- 데이터베이스/인증: Supabase (회원가입, 데이터 저장을 담당)

## ✍️ 코딩 스타일 (Vibe Rules)
1. **쉬운 주석:** 모든 함수와 복잡한 로직 위에는 한글로 "이 코드가 무엇을 하는지" 초등학생도 이해할 수 있게 설명해 주세요.
2. **간결한 구조:** 파일을 너무 쪼개지 마세요. 한 파일 안에서 프론트와 백엔드 로직(Server Actions)을 처리하는 것을 권장합니다.
3. **디자인 우선:** 비개발자에게 시각적 만족감은 중요합니다. shadcn/ui를 적극 활용해 현대적이고 깔끔한 UI를 먼저 만들어주세요.
4. **에러 가이드:** 에러가 발생하면 해결책만 주지 말고, "왜 이런 에러가 났는지"를 스터디원에게 과외 선생님처럼 친절하게 설명해 주세요.

## 📁 폴더 규칙 (최대한 단순하게)
- `app/`: 페이지와 기능 (Next.js 규칙)
- `components/`: 재사용할 버튼이나 입력창
- `lib/`: Supabase 설정 등 공통 도구
*그 외 폴더는 Claude가 판단해서 최소한으로 생성해 줘.*

## 🔐 보안 및 환경 변수 (Secrets)
1. **`.env.local` 보호:** 이 파일에는 Supabase 접속 키 등 민감한 정보가 들어있습니다. 절대로 GitHub에 업로드(Push)하지 마세요.
2. **`.gitignore` 확인:** 프로젝트 시작 시 `.env.local`이 `.gitignore`에 포함되어 있는지 반드시 확인하고, 누락되었다면 즉시 추가하세요.
3. **환경 변수 가이드:** 새로운 환경 변수가 필요할 때, Claude는 사용자에게 `.env.local`에 어떤 값을 추가해야 하는지 명확히 안내하고, 예시 파일(`.env.example`)을 만들어 관리하세요.
4. **키 노출 금지:** `SUPABASE_SERVICE_ROLE_KEY`와 같은 관리자용 키는 절대로 브라우저(Client Side) 코드에서 사용하지 않도록 엄격히 제한하세요.

## 🌿 Git & Branch Rules
- 우리 프로젝트의 기본(메인) 브랜치 이름은 **`main`**입니다.
- 모든 새로운 기능 개발과 배포의 기준은 `main` 브랜치를 사용하세요.
- 코드 변경 사항을 적용할 때는 반드시 `main` 브랜치에 있는 최신 코드를 확인한 후 진행하세요.
- 처음 배포 시 git의 사용자 이메일과, 클로드 코드의 사용자 이메일이 다를 수 있으므로 확인해주세요.
