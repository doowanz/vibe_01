## 🌐 웹서비스 구조 이해하기

1회차에서 만든 것은 **단순 HTML 파일**이었어요.
로그인, 데이터 저장 같은 기능을 추가하려면 조금 더 알아야 할 개념이 있어요.

### 📄 HTML(HyperText Mark-up Language)이란 무엇인가요?

**HTML은 문서예요.** 워드 파일이나 PDF처럼, 내용과 형식을 담은 파일이에요.
그리고 **브라우저(Chrome, Safari 등)는 그 문서를 읽어주는 뷰어**예요.

> 워드 파일을 열려면 Word가 필요하듯, HTML 파일을 열려면 브라우저가 필요해요.

1회차에서 만든 것도 이 HTML 문서예요. 브라우저만 있으면 어디서든 열 수 있어요.

### 백엔드는 왜 필요한가요?

HTML 문서만으로는 **브라우저 화면에 보여주는 것** 외에 아무것도 할 수 없어요.

| 기능 | HTML만으로? |
|---|---|
| 로그인 / 회원가입 | ❌ 안 됨 |
| 내가 쓴 글 저장하기 | ❌ 안 됨 |
| 다른 사람이 쓴 글 불러오기 | ❌ 안 됨 |
| AI에게 질문하고 답변 받기 | ❌ 안 됨 |
| 날씨·지도·번역 기능 가져오기 | ❌ 안 됨 |
| 결제 처리하기 | ❌ 안 됨 |
| 매일 밤 자동으로 리포트 보내기 | ❌ 안 됨 |

한마디로 백엔드는 **"사용자 눈에는 보이지 않지만, 서비스가 실제로 작동하게 만드는 모든 것"** 을 담당합니다.

구체적으로는 이런 일을 합니다.

**① 비즈니스 로직 처리** — "이 사람이 로그인할 자격이 있는가?", "할인 쿠폰이 유효한가?" 같은 판단과 규칙을 처리합니다.

**② 데이터 저장·조회** — 회원 정보, 게시글 등을 데이터베이스에 넣고 꺼내옵니다.

**③ 외부 API 호출** — OpenAI, 카카오맵처럼 다른 회사의 기능을 가져다 씁니다. API 키가 필요한 이유가 바로 여기에 있습니다.

**④ 보안 처리** — 비밀번호 암호화, 인증 토큰 발급 등 민감한 처리를 브라우저 밖에서 안전하게 수행합니다.

**⑤ 자동화 작업** — 특정 시간에 이메일 발송, 데이터 정리 등 사람이 직접 누르지 않아도 실행되는 작업을 처리합니다.

```
[프론트엔드]           [백엔드]                  [데이터베이스]
브라우저에 보이는 것  →  보이지 않는 모든 처리  →  데이터를 저장하는 곳
(HTML 문서)             · 비즈니스 로직              (Supabase 등)
                        · 보안·인증
                        · 자동화 작업
                             ↕
                        [외부 API]
                    기능을 빌려오는 곳
                  (OpenAI, 카카오맵 등)
```

> 백엔드는 식당으로 치면 **주방** 입니다. 손님(프론트엔드)은 메뉴만 보지만, 실제 요리·재료 관리·위생 처리는 모두 주방(백엔드)에서 일어납니다.

## 📋 벡엔드 구현을 위한 표준 규칙 주기을 AI에게 알려주기
백엔드는 프로그래밍 언어에 따라 달라져요.
스터디에서는 요즘 가장 많이 쓰는 방식으로 구현하기 위해 클로드에게 규칙을 알려줄거에요.
Claude Code는 프로젝트 폴더 안에 **CLAUDE.md** 파일이 있으면, 대화를 시작할 때마다 그 규칙을 자동으로 읽어요.
마치 신입 직원에게 사규(社規)를 건네주는 것과 같아요!

md(마크다운)이란? 마크업 보다 가볍게 사용하기 위해 만들어진 문서규격
무거운 것을 내려놓는 다는 의미에서 마크**업** -> 마크**다운**
```
HTML(마크업) : <strong>굵게</strong>
md(마크다운) : `**굵게**`
```

스터디 규칙 (CLAUDE.md) 주요내용
```
- Next.js + Tailwind CSS + Supabase를 사용해요
- 코드에 한글 주석을 달아줘요
- 보안이 필요한 키는 절대 공개된 곳에 올리면 안 돼요
- main 브랜치 기준으로 개발해요
```

### 🔑 API 키란?

---

#### 비유: "놀이공원 입장 팔찌"

| | 놀이공원 | API |
|---|---|---|
| 서비스 | 놀이기구 | AI 기능 |
| 입장권 | 팔찌 | API 키 |
| 무단 사용 시 | 요금 청구 | 비용 폭탄 |

---

### 3가지만 기억하세요

#### 1. 없으면 못 씁니다
API 키가 없으면 OpenAI 서버에 요청 자체가 불가합니다.

#### 2. 유출되면 내 돈 나갑니다
타인이 내 키로 사용한 비용은 키 소유자가 전액 부담합니다.

#### 3. 코드와 반드시 분리합니다

```js
// ❌ 코드에 직접 작성 — GitHub 업로드 시 전 세계 노출
const apiKey = "sk-proj-abcd1234..."

// ✅ 환경변수로 분리 보관
const apiKey = process.env.OPENAI_API_KEY
```

> 코드는 공개해도 됩니다. 키는 `.env.local` 금고에 따로 보관하세요.

장점: "이거 진짜 쓸 수 있겠다"는 실용성 체감, 공감대 폭발
단점: 텍스트 준비가 필요해서 실습 중 멈춤 포인트가 생길 수 있음

사용장소, 참석자, 주제를 입력하면 그럴듯한 회의록 작성

## 🚀 백엔드가 포함된 웹사이트 공개(배포)하기
시간이 되면 진행, github -> netlify 자동 배포
자동 배포 시 apikey 환경변수 설정 필요
자동 배포 시 아무나 사용할 수 없게 보안장치 필요 (고정 비밀번호를 쳐야 다음페이지로 넘어갈 수 있게 해줘)

### GitHub가 왜 필요한가요?

| | 1회차 방식 (직접 업로드) | 2회차 방식 (GitHub 연동) |
|---|---|---|
| 배포 방법 | 폴더를 매번 드래그 | 코드 저장 → 자동 배포 🚀 |
| 백엔드 포함 | ❌ 불가 | ✅ 가능 |
| 변경 기록 | ❌ 없음 | ✅ 히스토리 관리 |

### 실습 순서

**1단계. GitHub 가입**
1. [https://github.com](https://github.com) 접속
2. 회원가입 및 이메일 인증 완료

**2단계. Claude에게 GitHub 연결 요청**
```
github 연결해줘
github 사용 이메일은 [본인 이메일]이야
```

> Claude가 git 설정과 SSH 키 세팅을 자동으로 안내해줄 거예요!

**3단계. 소스코드 GitHub에 올리기**
```
소스 코드 github에 올려줘
```

> Claude가 `git init`, `commit`, `push`를 모두 처리해줘요!

**4단계. Netlify와 GitHub 연결하기**
1. [https://www.netlify.com](https://www.netlify.com) 로그인
2. **Add new site** → **Import an existing project** 클릭
3. **GitHub** 선택 → 내 저장소 선택 → **Deploy!**

> 이제 GitHub에 코드를 올리면 자동으로 배포돼요 🎉

배포된 사이트 URL로 접속해서 확인해보세요!
